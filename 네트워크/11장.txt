1. WAN 기술

    1. 교환 기술

        1. 교환기의 중계 기능 + 전송 기능 -> 데이터 : 목적지까지 전달함

        2. 교환망 : 다수의 교환기 + 전송 링크 구성

            1. 메쉬형과 유사한 토폴로지 -> 네트워크 구성
        
        3. 송신 스테이션 : 교환망으로 보낸 데이터

            1. 최종 : 수신 스테이션까지 전달
        
        전달 방식

            1. 전파 지연 : 신호가 한 교환기에서 다음 교환기로 도달하는데 걸리는 시간

            2. 전송 시간 : 데이터가 출발지에서 도착지까지 도달하는데 걸리는 시간

            3. 노드 지연 : 다음 경로로 전달되기까지 데이터가 교환기에서 대기하는 시간

            4. 데이터 처리율 : 일정 시간 동안 송수신할 수 있는 데이터 양

            종류

                1. 회선 교환 -> 물리적으로 회선을 설정한 후에 데이터를 전하는 방식에
                    
                    1. 음성 통신을 위해 개발

                    2. 데이터 전송 전 : 송신 스테이션 - 교환기 - 수신 스테이션

                        1. 물리적 회선 연결 -> 데이터 전송 방식

                            1. 회선 : 전송 링크 -> 시분할 , 주파수 분할 다중화

                            2. 하나의 채널에 해당 됨
                    
                        2. 고정적으로 각 링크마다 비어있는 회선에 할당

                    3. 연결 된 후 다음 접속이 해제될 때 까지 두 스테이션은 전용 회선 처럼 해당 회선 독점

                    4. Call 설정 -> 교환기 내에서 처리를 위한 지연은 없음

                        1. 데이터 : 확보된 채널 -> 일정 전송률로 전송

                    5. 정보 전송 완료 -> 해제 과정을 통해 점유한 회선 해제 -> 다른 통신을 위해 사용될 수 있도록 함

                        1. 실시간 전송이 요구되는 응용에 적합

                        2. 오류 없는 정보 전달이 요구되는 데이터 전송 -> 적합하지 않은 면이 있음

                    6. 일반적인 데이터 : On / Off 형태로 발생

                        1. 회선 교환 : 회선 사용률 -> 비효율적임

                        2. 실제 회선 사용률 : 10% 정도에 불과 -> 대부분 : 시간 낭비

                        3. 다양한 전송 속도를 요구하는 응용에 적합하지 않음

                        4. Call 설정 -> 네트워크 내 오류 제어 기능 제공 불가

                2. 축적 교환

                    1. 메시지 교환 -> 거의 사용하지 않음

                        1. 가변 길이의 메시지 단위로 데이터 교환하는 방식 -> 지정 - 전달 방식

                            1. 도착한 메시지 일단 저장

                            2. 다음 노드로 가는 링크 공백 -> 전송

                        2. 메시지 교환 : 송수신 스테이션 간 데이터 전송을 위한 전용 통신로 확보할 필요 없음

                        3. 링크를 동적으로 공유함 

                        동작 과정

                            1. 전송할 A 메시지 -> 설명 과정 없이 메시지 헤더에 목적지 주소 부착 -> 네트워크로 전송

                            2. 메시지 교환기 : 저장 장치를 가진 버용 컴퓨터 구현

                                1. 도착하는 A 메시지 : 저장 장치에 일단 저장

                                2. 교환기 : 메시지 A 완전 수신 -> 오류 검사

                                    1. 오류 O -> 경로 결정 : 다음 교환기로 메시지를 보냄

                                        1. 먼저 도착 -> 전송을 기다리는 메시지들은 모두 전송할 때 까지 대기 -> 이후 A 메시지 전송
                        
                        4. 메시지 전달 지연 : 비교적 길고 가변적임

                            1. 전자 우편, 파일 전송 등에 적합
                        
                        장점
                            
                            1. 링크를 효율적으로 이용할 수 있음

                            2. 전송 속도 변환 , 코드 변환 수행할 수 있음

                                1. 코드와 속도가 다른 스테이션끼리도 메시지 교환 가능

                        단점

                            1. 개선한 패킷 교환 기술 등장 -> 현재 메시지 교환 방식은 사용되지 않음

                    2. 패킷 교환

                        1. 패킷 단위로 전송 -> 저장 - 전달 방식과 유사함

                        2. 패킷 최대 길이 : 1000 바이트 정도로 제한됨

                        3. 각 패킷 : 네트워크 상의 경로 탐색 -> 목적지 까지 전달하기 위해 최소한의 정보가 들어 있음

                            1. 송수신 스테이션 주소

                            2. 전송하는 데이터 

                        종류

                            1. 가상 회선 방식 = 연결형 방식

                                1. 패킷을 전송하기 전 송수신 스테이션 간 회선 설정

                                2. 수신 측 : 전송된 순서대로 도착

                                    1. 회선을 구성하는 각각의 링크 -> 한 쌍의 송수신 스테이션에 독점하지 않음

                                        1. 다른 쌍의 송수신 스테이션과 공유될 수 있음
                                
                                교환 방식

                                    1. 경로 설정 : 통신 시작 전 송신자와 수신자 간 경로를 설정하고 패킷이 전송될 때마다 사용

                                    2. 패킷 라벨링 : 각 패킷에 가상 회선 번호 (VCI) 부여함

                                    3. 패킷 전송

                                        1. 송신자 : 데이터 -> 패킷으로 분할 -> VCI를 포함 시킴 -> 네트워크 전송

                                        2. 패킷 : 설정된 가상 회선을 따라 중간 노드를 거침 -> 수신자 전달

                                        3. 중간 노드 : VCI 사용 -> 각 패킷을 올바른 다음 노드로 전달함
                                    
                                    4. 순서 유지

                                    5. 연결 종료

                            2. 데이터그램 방식 = 비연결형 서비스 방식

                                1. 각 패킷별로 전송 경로가 선택되어 전송하는 방식

                                2. 모든 패킷의 헤더 -> 목적지의 완전한 주소가 들어 있어야 함

                                3. 전송된 패킷들은 순서가 뒤바뀌어 목적지에 도착할 수 있음

                                교환 방식

                                    1. 독립적인 패킷 처리 

                                        1. 송신자 : 데이터 -> 패킷으로 분할

                                        2. 각 패킷 : 목적지 주소 + 충분한 라우팅 정보 포함

                                        3. 패킷 -> 독립적으로 처리 -> 각각 개별적인 경로로 통해 목적지 전송

                                    2. 동적 라우팅 : 각 라우터는 도착하는 패킷 검사, 최적의 다음 경로 결정

                                        1. 네트워크 혼잡 상태, 링크의 실패와 같은 변화에 대응할 수 있음
                                    
                                    3. 비연결 지향적 : 데이터 전송 전 사전에 경로 설정 X -> 독립적으로 목적지를 향해 이동

                                        1. 중간 라우터 : 각 패킷을 개별적으로 처리

                                    4. 패킷의 순서, 무결성

                                        1. 도착 순서 : 원본 데이터의 순서와 다를 수 있음

                                        2. 수신자 측 : 도착하는 패킷 재조립 -> 순서대로 재정렬

                                        3. 패킷 손실, 오류 발생 -> 오류 복구 매커니즘을 통해 처리

                                    5. 네트워크 혼잡, 부하 분산 : 특정 경로의 과부하를 피할 수 있음

2. 이동 통신망

    1. 셀룰러 이동 통신 시스템 구성

        1. 서비스 지역 -> 여러 개의 셀로 분할 -> 각 셀에 고유한 주파수 이용

            1. 상호 간섭을 주지 않을 만큼 떨어진 지역 -> 동일한 주파수를 재사용함
        
        2. 셀룰러 이동 통신망의 특징

            1. 핸드 오프 : 통화를 끊김 없이 계속 유지하기 위해 이동한 기지국 또는 셀로 바꾸어 주는 기술

                종류

                    1. 기지국 간 통화 채널 전환

                        1. 소프트 핸드오프

                            1. 새로운 셀의 채널 먼저 연결

                            2. 현재 셀의 채널 -> 나중에 끊음

                            3. 복수의 기지국 -> 동일한 주파수일 때만 가능

                                1. CDMA 방식 : 같은 주파수 대역을 사용하기 때문에 쉽게 구현 가능

                        2. 하드 핸드오프
                            
                            1. 주파수 할당이 변하거나 이동 통신 교환기가 변경될 때 발생하는 핸드오프
                            
                            2. 현재 통화 중인 채널 끊음 -> 곧바로 다른 채널로 연결

                            3. 순간적으로 통화의 끊김 발생 -> 통화에서 이를 느끼지 못함

                            4. FDMA, TDMA 방식에서 주로 사용

                    2. 기지국 내 통화 채널 전환

                        1. 소프터 핸드오프

                            1. 일반적으로 도심 기지국은 3섹터로 구성 -> 각 섹터의 안테나는 120도 씩 커버함

                            2. 동일 주파수 + 동일한 셀 내에서 섹터 간에 발생하는 핸드오프

                            3. 같은 기지국의 섹터 간 전파가 겹지는 지역 -> 2개의 섹터를 통하여 통화가 이루어지는 과정

                                1. 소프트 핸드오프 방식과 유사한 절차를 따름

        3. 셀룰러 망의 장점

            1. 넓은 커버리지 : 이동 중에도 연결 유지 -> 도시 뿐만 아니라 교외 지역에서도 통신 서비스를 제공할 수 있음

            2. 고속 데이터 전송 : 초고속 인터넷 연결 가능 -> 스트리밍, 비디오 회의 등을 원활하게 지원

            3. 이동성 : 사용자가 이동 중에 전화 통화, 인터넷 사용, 메시지 전송 등을 할 수 있게 해줌

            4. 확장성 : 네트워크 운영자 수요의 증가에 따라 서비스를 확장할 수 있음을 의미함

            5. 다양한 서비스 제공

            6. 보안 : 데이터와 통화의 안전 보장

            7. 비상 서비스 접근 : 사용자가 어디에서나 긴급 서비스에 연락 가능

            8. 지속적인 기술 발전 : 더 빠른 속도 , 더 나은 연결성, 새로운 기능들을 가능하게 함

        4. 전력 제어 

            1. 신호 품질 유지 : 통화 품질 보장 + 데이터 전송의 신뢰성을 높이는 데 필수적임

            2. 인터퍼런스 최소화 : 여러 사용자가 동일한 주파수 대역 공유

                1. 사용자 간 간섭 최소화 -> 각 사용자의 전송 전력을 조절함

            3. 배터리 수명 연장 : 필요 이상의 전력을 사용하지 않음 

            4. TDMA 시스템 : 간섭 증가 정도의 영향을 미침

            5. CDMA 시스템 : 채널 용량에 심각한 영향을 줌

                1. 수신 전력이 같을 때 : 최대 채널 용량을 달성하려 함

                    1. 전력 차이가 크면 멀리 있는 이동국이 가까운 이동국의 간섭 발생

                        1. 복조 불가능 -> 근거리 / 원거리 문제 발생
                    
                2. 문제 : CDMA가 간섭에 의해 채널 용량이 결정되는 시스템이라 나타남

                    1. 해결 책 : 기지국에 수신되는 각 이동국의 수신 전력이 일정하도록 이동국의 송신 전력 조정 
            
            전력 제어 목적

                1. 기지국 통화 용량의 최대화

                2. 이동국 배터리 수명 연장

                3. 각 사용자 간 통신 품질의 공평성 보장

                4. 양호한 통화 품질 유지
            
            특징

                1. 기지국에서 멀어질수록 단말기 -> 송신하는 신호 세기가 약해짐

                2. 단말기의 송신 신호 강해짐 -> 자신의 통화 품질은 좋아짐

                    1. 다른 단말기에 간섭을 줌 -> 다른 단말기의 통화 품질이 나빠짐
                
                3. 단말기 간 혼선을 방지하는 방법

                    1. 통화 채널을 통해 8단계 적정 출력 범위 내 자동으로 적정 출력 유지
