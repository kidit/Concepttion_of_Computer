1. 다중화

    1. 다중화의 개념

        1. 단일 링크로 신호 여러 개를 동시에 전송할 수 있는 기술

        2. 입력 회선 (N 개) -> 다중화기 -> 1개로 통합 -> 다중 복구기 -> 여러개 분할 -> 출력 회선 (N 개)
    
    2. 다중화 방식

        1. FDM -> 아날로그 신호 다중화

            1. 주파수 분할 다중화

                1. 직교 주파수 분할 다중화 (OFDM)

                    1. 스펙트럼 중심부 : 주변 채널의 간섭이 일어나지 않도록 함 -> 상당한 주파수 절약

                    2. 일정한 간격으로 떨어져 있는 여러 개의 부반송파 -> 데이터 분산 -> 전송 = 멀티 캐리어 전송 방식

                    장점
                    
                        1. 다중 경로 페이딩에 강함

                        2. 고속 전송 가능

                        3. 이동 통신 -> 셀 간 간섭이 없음

                        4. 자원 할당 용이

                        5. WiMAX, LTE -> 이동 통신 분야에서도 사용됨

            2. 변조된 신호 -> 하나의 복합 신호로 합침 -> 큰 대역폭의 링크를 통해 전송

            3. 다중 복구기 -> 대역 통과 필터 사용 : 수신된 복합 신호 분리

            4. 복조기 -> 반송파에서 신호만 분리 -> 수신 장치로 보냄

            5. 아날로그 계층 구조

                1. 그룹 형성화 -> 슈퍼 그룹으로 다중화 -> 마스터 그룹으로 다중화 -> 점보 그룹으로 다중화
            
            6. 기술 적용 예

                1. 주파수 분할 다중화

                    1. 구식 아날로그 전화망

                    2. 아날로그 이동 전화 (AMPS)
                
                2. 직교 주파수 분할 다중화

                    1. WiBro, WiMAX

                    2. LTE, LTE-A 
        
        2. TDM -> 디지털 신호 다중화

            1. 시분할 다중화 : 시간을 공유함 -> 높은 대역폭을 갖는 링크 1개 -> 여러 회선이 공유하는 디지털 다중화 방식

            2. 동기 TDM, 비동기 TDM

                1. 동기 TDM

                    1. 전송 매체의 데이터 전송률이 전송할 디지털 신호의 데이터 전송률보다 높을 때 사용하는 방법

                    2. 각 장치에 할당된 시간이 돌아옴 -> 해당 장치는 데이터 일부를 보냄

                        1. 만일 전송할 데이터가 없을 때 -> 해당 타임 슬롯은 빈 상태로 전송

                    3. 타임 슬롯 -> 프레임으로 묶임

                        1. 한 장치에 주어진 타임 슬롯 -> 각 프레임에서 매번 같은 위치를 차지하게 됨 -> 해당 장치의 채널 구성

                    4. 인터리빙 : 일정한 속도와 정해진 순서로 회전하는 과정

                    5. 비트 삽입식, 문자 삽입식

                        1. 비트 삽입식

                            1. 타임 슬롯 크기 -> 비트 단위로 이루어짐

                            2. 버퍼의 크기가 작아도 됨
                        
                        2. 문자 삽입식

                            1. 타임 슬롯의 크기 -> 문자 단위

                            2. 수신 측에서 문자를 재구성하기 위한 오버헤드가 필요 없음
                        
                    단점 : 각 입력 회선에 고정적으로 타임 슬롯을 할당

                        1. 송신 장치가 데이터를 보내지 않으면 해당 타임 슬롯이 낭비됨
                        
                        해결책

                    6. 송수신 간 동기화 -> 각 프레임 앞부분에 프레임 비트를 추가함

                        1. 0,1을 교대 -> 매 프레임에 하나 씩 추가

                        2. 다중 복구기 : 해당 패턴을 통해 타임 슬롯을 정확하게 분리 -> 동기화 문제 해결

                        3. 프레임 비트 -> 전송 효율을 떨어뜨리는 오버헤드가 될 수 있음

                        4. 데이터 전송률이 다른 장치들은 하나의 동기 TDM에서 수용 가능

                            1. 비트 채우기 : 다중화기가 여분의 비트를 추가해 넣는 것

                                1. 정수배가 되지 않을 때 사용

                                2. 추가된 여분의 비트 -> 다중 복구기에 의해 분리
                    
                    7. 입력과 출력이 이미 고정됨 -> 주소 필요 X
                
                2. 비동기 TDM

                    1. 동기 TDM의 타임 슬롯 낭비 요인 제거 목적

                    2. 실제 전송 데이터가 있는 장치에만 타임 슬롯 할당

                    3. 전송할 데이터가 없을 때 -> 타임 슬롯 할당 X  -> 주소가 반드시 필요함

                    4. 동작과정

                        1. 각 송신 장치에 데이터 발생
                            
                            1. 지정된 프레임 크기만큼 데이터 저장

                            2. 각 장치의 프레임 주소와 함께 임시 버퍼에 저장

                        2. 버퍼에 저장된 데이터 프레임 -> 타임 슬롯 할당 -> 전송 -> 반복 : 다중화 수행

                        3. 수신부 : 수신된 프레임 주소 -> 프레임 분리
                    
                    5. 같은 회선을 사용 시 동기 TDM보다 더 많은 장치 지원 

                    6. 프레임의 타임 슬롯 개수는 한꺼번에 전송할 가능성이 있는 입력 회선의 개수에 대한 통계적 분석에 기초함

                        1. 전체 링크 용량을 100% 사용하지 않을 수 있음

                        2. 타임 슬롯을 동적으로 할당 -> 낭비 가능성과 정도가 크게 줄어듦
                    
                    7. 수신 주소를 타임 슬롯에 넣어야 함
            
            3. 기술 적용 예

                1. 시분할 다중화

                    1. 전화망 (FSTN)

                    2. 전용 회선

                        1. T-1 , E-1
                    
                    3. 동기 디지털 계위 전송망 (SDH)

                    4. 유럽 디지털 이동 통신 (GSM)
                
                2. 통계적 시분할 다중화

                    1. LAN

                    2. 인터넷

                    3. 초고속 국가망 (ATM, 프레임 릴레이)

        3. CDM -> FDM + TDM = 확산 대역을 이용한 다중화

            1. 코드 분할 다중화

                1. 다수 사용자 
                
                    1. 주파수와 시간을 공유 

                    2. 각 사용자에게 서로 다른 코드를 부여하는 신호 * 코드 -> 전송

                    3. 수신 : 송신 시 사용된 코드 * 동일 코드 = 원래 신호 복구 (FDM) + (TDM) 혼합 -> 다중화 방식

                    4. 특징

                        1. 각 사용자에게 할당되는 코드 : 상호 직교 관계에 있음 -> 영향을 미치지 않음

                        2. CDM -> 사용자 간 간섭에 의해 시스템 용량 제한 -> 서로 영향을 미치지 않는 코드 할당 -> 다중화 : 전력 제어 필요

                        3. 자신과 같은 코드를 곱합 = 1

                        4. 자신과 다른 코드를 곱함 = 0

                        5. 동일한 주파수로 동시 접속 가능

                        6. 대역 확산 기술 사용 
                        
                            1. 잡음의 영향이 좁은 대역 : 신호 -> 큰 영향

                            2. 상대적으로 대역 확산된 신호 : 신호 -> 큰 영향을 받지 않음 
                        
                        7. 코드 -> 난수 생성 알고리즘 이용 -> 임의의 패턴을 갖는 2진수 열로 나타냄

                            1. 주파수 대역 확산 : 다양한 반송 주파수 사용 -> 반송 주파수를 모두 합한 주파수 대역으로 확산

                            2. 도약 패턴 : 슈도랜덤 코드 발생기 -> 난수 발생 알고리즘에 의해 결정

                            3. 직접 순열 대역 확산 : 칩이라는 비트 코드를 사용

                                1. 칩 : 데이터의 각 비트에 곱해져 데잍 비트율을 n 배 증가시킴 = 변조되는 신호의 대역폭도 n 배 증가 (예 : 바커 순열)

                2. 기술 적용 예

                    1. 코드 분할 다중화

                        1. 국내 디지털 이동 통신

                        2. 군용 무선 통신 (미국)
